### 一、题目描述
在一条环路上有 n 个加油站，其中第 i 个加油站有汽油 gas[i] 升。

你有一辆油箱容量无限的的汽车，从第 i 个加油站开往第 i+1 个加油站需要消耗汽油 cost[i] 升。你从其中的一个加油站出发，开始时油箱为空。

给定两个整数数组 gas 和 cost ，如果你可以按顺序绕环路行驶一周，则返回出发时加油站的编号，否则返回 -1 。如果存在解，则 保证 它是 唯一 的。

示例 1:
> 输入: gas = [1,2,3,4,5], cost = [3,4,5,1,2]
>
> 输出: 3
>
>解释:
>
> 从 3 号加油站(索引为 3 处)出发，可获得 4 升汽油。此时油箱有 = 0 + 4 = 4 升汽油
>
>   开往 4 号加油站，此时油箱有 4 - 1 + 5 = 8 升汽油
>
>   开往 0 号加油站，此时油箱有 8 - 2 + 1 = 7 升汽油
>
>   开往 1 号加油站，此时油箱有 7 - 3 + 2 = 6 升汽油
>
>   开往 2 号加油站，此时油箱有 6 - 4 + 3 = 5 升汽油
>
>   开往 3 号加油站，你需要消耗 5 升汽油，正好足够你返回到 3 号加油站。
>
>   因此，3 可为起始索引。

示例 2：
> 输入: gas = [2,3,4], cost = [3,4,3]
>
>输出: -1
>解释:
>
>你不能从 0 号或 1 号加油站出发，因为没有足够的汽油可以让你行驶到下一个加油站。
>
>我们从 2 号加油站出发，可以获得 4 升汽油。 此时油箱有 = 0 + 4 = 4 升汽油
>
>开往 0 号加油站，此时油箱有 4 - 3 + 2 = 3 升汽油
>
>开往 1 号加油站，此时油箱有 3 - 3 + 3 = 3 升汽油
>
>你无法返回 2 号加油站，因为返程需要消耗 4 升汽油，但是你的油箱只有 3 升汽油。
>
>因此，无论怎样，你都不可能绕环路行驶一周。

提示:
- n == gas.length == cost.length
- 1 <= n <= 105
- 0 <= gas[i], cost[i] <= 104
- 输入保证答案唯一。


### 二、解题思路
#### 思路 1：暴力解法（易理解，但效率低）
**核心逻辑**
遍历每个加油站作为「候选起点」，模拟绕圈过程，检查是否能走完一周。

**步骤**
- 遍历每个索引 i（0 ~ n-1）作为起点；
- 初始化当前油箱油量 curr_gas = 0，模拟行驶 n 步（覆盖环形所有加油站）；
- 对每一步 k，计算当前加油站 j = (i + k) % n，curr_gas += gas[j] - cost[j]；
- 若中途 curr_gas < 0，说明从 i 出发失败，直接跳出，尝试下一个起点；
- 若遍历完 n 步后 curr_gas ≥ 0，返回 i；
- 所有起点都尝试失败，返回 -1。

**复杂度** 
- 时间复杂度：O(n^2)，其中 n 为数组长度。每个起点最多遍历 n 步。
- 空间复杂度：O(1)，仅使用了常数个额外变量。

> 暴力解法在 LeetCode 中会超时（n=1e5 时无法通过），仅用于理解题目本质。

#### 思路 2：贪心解法（最优，O (n) 时间）
**核心逻辑**
- 若从起点 start 出发，走到 i 时油箱油量为负，说明 start ~ i 之间的所有点都不可能是有效起点（因为从这些点出发，到 i 时油量只会更少）；
- 此时直接将候选起点更新为 i+1，并重置当前油箱油量，继续遍历；
- 遍历结束后，因已满足 sum(gas) ≥ sum(cost)，最终的 start 就是唯一解。

**步骤拆解**
- 初始化变量：
    - total_gas：统计所有加油站的总汽油量；
    - total_cost：统计绕一圈的总耗油量；
    - curr_gas：当前油箱的实时油量；
    - tart：候选起点（初始为 0）。
- 遍历每个加油站 i：
    - 累加 total_gas += gas[i]、total_cost += cost[i]；
    - 计算当前油量变化：curr_gas += gas[i] - cost[i]；
    - 若 curr_gas < 0：说明从 start 到 i 无法通行，更新 start = i+1，并重置 curr_gas = 0。
- 最终判断：
    -若 total_gas ≥ total_cost，返回 start；否则返回 -1

### 三、题目实现
```javascript
var canCompleteCircuit = function(gas, cost) {
    // 边界处理：非数组/长度不一致直接返回-1
    if(!Array.isArray(gas) || !Array.isArray(cost) || gas.length !== cost.length) {
        return -1;
    }

    const n = gas.length;
    let totalGas = 0;    // 总汽油量
    let totalCost = 0;   // 总耗油量
    let currGas = 0;     // 当前油箱油量
    let start = 0;       // 候选起点

    for(let i = 0; i < n; i++) {
        totalGas += gas[i];
        totalCost += cost[i];
        // 核心：计算从i到i+1后的剩余油量（正确的油量变化逻辑）
        currGas += gas[i] - cost[i];

        // 贪心关键：当前油量不足，说明start~i都无效，起点更新为i+1
        if(currGas < 0) {
            start = i + 1; // 修正：更新为i+1，而非置0
            currGas = 0;   // 重置当前油量
        }
    }

    // 总油量足够则返回start，否则返回-1
    return totalGas >= totalCost ? start : -1;
};
```
